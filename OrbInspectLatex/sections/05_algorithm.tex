\section{Offline Planner}
\label{sec:algorithm}

\subsection{Decomposition}

The implemented planner decomposes inspection into mesh target sampling,
candidate viewpoint construction, visibility matrix construction, viewpoint
selection, CW tour generation, trajectory rollout, and camera attitude export.
This decomposition is deliberately transparent: each stage writes artifacts
that can be checked after the run. The self-contained benchmark includes
method-comparison, viewpoint, planner, trajectory, attitude, and coverage CSV
files.

\subsection{Candidate Viewpoints}

For each mesh target, candidate viewpoints are generated on offset shells and
accepted only when they satisfy basic safety and view constraints. Each
candidate $z_j$ stores a safety margin and a precomputed set of visible
targets. The resulting binary visibility relation can be written as
\begin{equation}
  A_{ij}=1
  \quad \Longleftrightarrow \quad
  \text{target } i \text{ is visible from candidate } z_j .
\end{equation}
In the high-coverage offline benchmark, 357 candidates are generated for
180 mesh target samples.

\subsection{Dynamics-Aware Viewpoint Selection}

The proposed method first performs dynamics-aware weighted set cover and then
orders the selected candidates with a greedy CW-transfer tour. At each set-cover
iteration, the planner balances new surface area coverage against the CW
transfer cost from the current predicted state. For candidate $z_j$, let
$\Delta A_j$ be the newly visible mesh area and let $J_{\mathrm{CW}}(x,z_j)$ be
the transfer cost defined in Section~\ref{sec:problem}. The set-cover stage
uses the score
\begin{equation}
  s_j =
  \frac{\Delta A_j}{1+J_{\mathrm{CW}}(x,z_j)}
  + w_m \min\{\widehat{d}_{\min}(x,z_j),d_{\max}\},
\end{equation}
then updates the predicted state with the CW rollout to the selected
viewpoint. A second ordering pass greedily chooses the next selected candidate
with the lowest predicted CW dynamic cost from the current terminal state. This
makes later choices depend on the orbital state reached by earlier choices, so
CW dynamics affect both the selected viewpoints and their order. The logged
\texttt{planner.csv} file records the selected candidate
identifier, new target count, area gain, cumulative coverage, score, transfer
$\Delta v$, dynamic transfer cost, minimum clearance, transfer tracking error,
and feasibility flag for each selected viewpoint.

\subsection{CW Transfer Generation}

After a candidate is selected, the planner computes a bounded-acceleration
CW transfer toward the viewpoint, samples the resulting trajectory, evaluates
clearance and speed, and updates predicted coverage. The exported trajectory
includes position, velocity, acceleration command, camera boresight, yaw,
pitch, and quaternion samples, so the planned path specifies both where the
chaser flies and how the inspection camera is pointed. Edges that violate
basic feasibility checks are penalized or rejected depending on the baseline
method. The current implementation is still a tractable approximation rather
than a global mixed-integer optimum; this limitation is discussed in
Section~\ref{sec:discussion}.

\subsection{Outputs}

The planner outputs a timestamped reference trajectory
\begin{equation}
  \mathcal{P} =
  \{(t_k,r_k^{\mathrm{ref}},v_k^{\mathrm{ref}},u_k^{\mathrm{ref}})\}_{k=0}^{K}.
\end{equation}
For the reported set-cover CW tour run, $\mathcal{P}$ contains 920 trajectory
samples over 1840~s with an 80~s transfer duration and 2~s integration step.
The trajectory and attitude stream are subsequently available to the ROS stack
for replay, logging, RViz inspection, and Gazebo visualization.
