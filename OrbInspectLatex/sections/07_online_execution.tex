\section{Online Execution}
\label{sec:online}

The present paper focuses on offline planning, but an inspection framework
must still define how the offline reference would be executed. We therefore
include the online layer as an interface specification rather than as a
separate contribution. Its purpose is to track the planned trajectory, enforce
runtime safety margins, and trigger replanning when the estimated state no
longer satisfies the offline assumptions.

\subsection{Reference Tracking}

The offline planner outputs a reference sequence
\begin{equation}
  \mathcal{P}=\{(t_k,r_k^{\mathrm{ref}},v_k^{\mathrm{ref}},u_k^{\mathrm{ref}},
  q_k^{\mathrm{ref}})\}_{k=0}^{K}.
\end{equation}
Online execution tracks this sequence using the current state estimate
$\hat{x}_k$. The nominal control is generated by a tracking law of the form
\begin{equation}
  u_k^{\mathrm{nom}} = u_k^{\mathrm{ref}} + K_x(\hat{x}_k - x_k^{\mathrm{ref}}),
\end{equation}
where $K_x$ is a stabilizing gain chosen for the HCW linearization. This layer
does not define the planner. It realizes the offline reference in closed loop
and provides the state and command streams used for verification.

\subsection{Safety Filter}

To preserve the keep-out constraint online, the nominal command can be passed
through a control-barrier-function (CBF) filter:
\begin{align}
  u_k^{\mathrm{exec}} = \arg\min_{u \in \R^3}\quad & \norm{u-u_k^{\mathrm{nom}}}_2^2 \\
  \mathrm{s.t.}\quad &
  h(x_k) \ge 0, \nonumber\\
  & \dot{h}(x_k,u) + \alpha h(x_k) \ge 0, \nonumber\\
  & \norm{u}_2 \le u_{\max}. \nonumber
\end{align}
Here $h(x_k)=d_{\mathcal S}(r_k)-d_{\mathrm{safe}}$ and $\alpha>0$ is a
barrier gain. If the QP is infeasible or the tracking deviation exceeds a
predefined threshold, the execution layer requests replanning from the offline
planner. The CBF layer gives a compact runtime safety constraint, but it is
treated here as an execution safeguard rather than as the offline planning
algorithm.

\subsection{Trigger Conditions}

Replanning is event-triggered rather than continuous. A new offline plan is
requested when one of the following occurs:
\begin{enumerate}
  \item the tracking error exceeds a tolerance,
  \item the barrier filter remains active beyond a prescribed duration,
  \item the executed coverage falls behind its predicted envelope,
  \item the state estimator reports loss of consistency.
\end{enumerate}
This event logic keeps the online layer lightweight while preserving a clear interface
between planning, tracking, and replanning. In the manuscript, the scientific
contribution remains the offline planner; the online layer defines how that
plan would be monitored during execution.

\subsection{Camera-Model Verification}

The replay node also serves as a geometric verifier for the camera model.
Using the saved attitude stream, it reconstructs the camera frame, publishes
the boresight marker, and generates the camera frustum from the same field of
view parameters used during planning. The ROS layer therefore verifies that
the offline camera geometry can be instantiated consistently at runtime.
The corresponding paper figure shows the frustum, selected SOOA terminal
poses, and visible targets together with the replayed trajectory.
